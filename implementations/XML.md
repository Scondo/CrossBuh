XML - универсальный формат хранения данных.
Также протокол SOAP использует XML-данные внутри контейнера в качестве запроса или результата.

Скаларные данные хранятся как значения тегов в записи соответствующих типов xsd
* decimal для целых и действительных чисел; 
* string для строк
* base64Binary для строк данных
* boolean для логических значений
* dateTime или date - для даты (если допустимо - без времени)

* Структуры передаются как множество тегов, где имя тега соответствует имени поля, а значение тега - значению поля
{поле1: переменная1; поле2: переменная2} записывается как <поле1>переменная1</поле1><поле2>значение2</поле2>

* Массивы передаются как повторяющийся тег item.
[переменная1, переменная2, переменная3] записывается как <item>перменная1</item><item>перменная2</item><item>перменная3</item>

Если массив является элементом структуры, то в качестве имени тега используется имя соответствующего поля
{поле0: переменная0; поле1:[переменная1, переменная2, переменная3]} записывается как <поле0>переменная0</поле0><поле1>перменная1</поле1><поле1>перменная2</поле1><поле1>перменная3</поле1>

Имя тега также может явно задаваться в описании формата, хотя это настоятельно не рекомендуется из-за снижения переносимости.
